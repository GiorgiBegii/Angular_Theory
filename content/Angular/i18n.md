# i18n

i18n means internationalization: preparing an app for multiple languages and locales.

It includes:

- translated text
- date/number/currency formats
- pluralization
- locale switching
- right-to-left layout when needed

## Angular i18n

Angular has built-in i18n support.

It is good for:

- compile-time translations
- static text
- strong framework integration
- apps with stable translation sets

Tradeoff:

- runtime language switching is less flexible than some libraries

## Transloco

Transloco is a popular Angular translation library.

It is good for:

- runtime language switching
- lazy-loaded translation files
- feature-scoped translations
- dynamic translation keys
- simpler app-level translation workflows

Use libraries like Transloco when the app needs flexible runtime translation behavior.

## Translation Keys

Good keys are stable and meaningful.

Example:

```json
{
  "users.list.title": "Users",
  "users.list.empty": "No users found"
}
```

Avoid keys based only on English text if the text changes often.

## Structure

Common structure:

```text
i18n/
+-- en.json
+-- ka.json
+-- de.json
```

Feature structure:

```text
users/
+-- i18n/
|   +-- en.json
|   +-- ka.json
```

Feature-scoped translation files help large apps stay organized.

## Best Practices

- use consistent key naming
- group keys by feature/domain
- support pluralization properly
- format dates/numbers/currency by locale
- avoid hardcoded UI text
- avoid building sentences by concatenating translated fragments
- keep translation ownership clear

## Short Answer

i18n organizes an app for multiple languages and locales. Angular built-in i18n is strong for compile-time/static translations. Transloco is more flexible for runtime language switching and lazy-loaded translation files. In large apps, maintainable i18n depends on stable keys, feature-based translation structure, pluralization, locale formatting, and avoiding hardcoded UI text.
